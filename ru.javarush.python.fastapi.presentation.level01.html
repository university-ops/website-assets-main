Уровень 1: Введение в FastAPI

Что такое FastAPI и почему он важен

FastAPI — это современный, высокопроизводительный веб-фреймворк на Python для создания API и микросервисов.

Он построен на основе асинхронной архитектуры ASGI (Asynchronous Server Gateway Interface), что позволяет ему эффективно обрабатывать большое количество одновременных подключений. Это делает его идеальным выбором для высоконагруженных систем, real-time приложений (чаты, уведомления) и микросервисной архитектуры.

Ключевые особенности:

  • один из самых быстрых фреймворков Python: сопоставим с NodeJS и Go.
  • встроенная поддержка async и await для неблокирующих операций.
  • использует стандартные аннотации типов Python и библиотеку Pydantic для валидации данных, что уменьшает количество ошибок.
  • генерирует интерактивную документацию API (Swagger UI / OpenAPI и ReDoc) прямо из вашего кода.
  • минималистичный синтаксис и автодокументация значительно ускоряют процесс создания API.

Чтобы понимать, почему FastAPI так эффективен, важно видеть его архитектуру в целом. Процесс обработки одного запроса можно представить как конвейер, где каждый компонент выполняет четкую задачу.

Сравнение FastAPI с Django и Flask

Критерий Django Flask FastAPI
Тип монолитный фреймворк ("батарейки в комплекте") микрофреймворк (минималистичный) микрофреймворк (с фокусом на API)
Основное применение крупные веб-приложения, сайты с админкой простые сайты, прототипы, гибкие проекты высокопроизводительные API, микросервисы
Асинхронность (ASGI) поддерживается, но не является основной парадигмой требует дополнительной настройки (Flask 2.0+) встроена в ядро, основное преимущество
Валидация данных Django Forms / DRF Serializers требует сторонних библиотек (Marshmallow, WTForms) встроена через Pydantic (строгая типизация)
Документация API требует DRF или сторонних библиотек требует сторонних библиотек (Flasgger) автоматическая (Swagger UI, ReDoc) "из коробки"

Django хорош для монолитных сайтов. Flask дает максимальную гибкость. FastAPI — лучший выбор для создания современных, быстрых и надежных API, благодаря встроенной асинхронности, валидации данных и автодокументации.

Установка FastAPI и Uvicorn

FastAPI — это сам фреймворк. Для его запуска необходим ASGI-сервер, который будет обрабатывать асинхронные запросы. Самый популярный сервер для FastAPI — это Uvicorn.

Шаги установки (рекомендуется в виртуальном окружении):

  1. Создание и активация виртуального окружения: 
    python -m venv venv
source venv/bin/activate  # macOS/Linux
# venv\Scripts\activate    # Windows
  2. Установка FastAPI: 
    pip install fastapi
  3. Установка Uvicorn:

    Пакет "uvicorn[standard]" включает дополнительные зависимости для лучшей производительности.

    pip install "uvicorn[standard]"
  4. Сохранение зависимостей: 
    pip freeze > requirements.txt

Настройка проекта и структура приложения FastAPI

Для FastAPI не существует строгой, обязательной структуры проекта, как в Django. Однако для поддержания порядка рекомендуется следовать определенным соглашениям.

Минимальная структура:

my_project/
└── main.py

В файле main.py создается экземпляр FastAPI и определяются эндпоинты.

Рекомендуемая структура для средних и крупных проектов:

my_project/
├── app/                  # Основная папка приложения
│   ├── __init__.py
│   ├── main.py           # Главный файл, собирающий приложение
│   ├── core/
│   │   └── config.py     # Настройки
│   ├── routers/
│   │   ├── __init__.py
│   │   └── posts.py      # Маршруты для постов
│   │   └── users.py      # Маршруты для пользователей
│   ├── schemas/
│   │   └── posts.py      # Pydantic модели (схемы)
│   └── crud/
│       └── posts.py      # Функции для работы с БД
├── tests/                # Папка с тестами
└── .env                  # Переменные окружения

В main.py роутеры из папки routers/ подключаются к основному приложению с помощью app.include_router(). Это делает код модульным и читаемым.

Создание первого простого API (GET-запрос)

В FastAPI эндпоинты создаются с помощью декораторов операций пути (Path Operation Decorators).

Для обработки GET-запроса используется декоратор @app.get().

Пример (main.py):

from fastapi import FastAPI

# 1. Создаем экземпляр приложения FastAPI
app = FastAPI()

# 2. Определяем эндпоинт для GET-запроса по корневому URL ("/")
# Эта функция будет вызываться, когда кто-то зайдет на http://127.0.0.1:8000/
@app.get("/")
def read_root():
    # 3. Возвращаем словарь, который FastAPI автоматически преобразует в JSON
    return {"message": "Hello, World!"}

# Другой эндпоинт
@app.get("/items/")
def get_items():
    return {"items": ["Яблоко", "Банан", "Апельсин"]}

Запуск сервера. В терминале, находясь в папке с main.py, выполните:

uvicorn main:app --reload
  • main: имя файла main.py.
  • app: имя переменной с экземпляром FastAPI внутри файла.
  • --reload: сервер будет автоматически перезапускаться при изменениях в коде.

Концепция эндпоинтов

Маршрутизация (Routing) в FastAPI — это процесс сопоставления URL-адреса и HTTP-метода запроса с конкретной функцией-обработчиком (эндпоинтом).

Path Operations

FastAPI предоставляет декораторы для всех основных HTTP-методов:

  • @app.get()
  • @app.post()
  • @app.put()
  • @app.delete()
  • @app.patch(), @app.options(), @app.head(), @app.trace()

Path Parameters.
Вы можете определять динамические части в URL, используя фигурные скобки. Значения этих частей передадутся в вашу функцию как аргументы. FastAPI автоматически валидирует их тип на основе аннотаций типов Python.

@app.get("/items/{item_id}")
def read_item(item_id: int): # FastAPI проверит, что item_id — это число
    return {"item_id": item_id, "description": f"Это товар номер {item_id}"}

Запрос: GET /items/123 -> item_id будет 123 (тип int).
Запрос: GET /items/abc -> FastAPI вернет ошибку 422 Unprocessable Entity.

Query Parameters

Query Parameters — это параметры, передаваемые в URL после знака ? в виде пар ключ=значение. Они используются для фильтрации, сортировки или пагинации.

Пример URL: /items/?skip=0&limit=10

В FastAPI Query-параметры определяются как аргументы функции, которые не являются частью пути.

Пример:

from fastapi import FastAPI, Query
from typing import Optional, List # Для опциональных параметров и списков

app = FastAPI()

# Базовый пример с опциональным параметром
@app.get("/search")
def search_items(query: Optional[str] = None): # 'query' — query-параметр
    if query:
        return {"message": f"Поиск по запросу: '{query}'"}
    return {"message": "Покажите все элементы"}

# Пример с валидацией и значениями по умолчанию
@app.get("/products")
def get_products(
    is_available: bool = True, # Обязательный (т.к. нет Optional), по умолчанию True
    limit: int = Query(10, gt=0, le=100), # Валидация: limit > 0 и <= 100
    tags: Optional[List[str]] = Query(None) # Список query-параметров
):
    # URL: /products/?tags=new&tags=sale
    # tags будет ['new', 'sale']
    return {"limit": limit, "is_available": is_available, "tags": tags}

Использование Query() из fastapi позволяет добавлять валидацию (больше, меньше, длина строки и т.д.) и метаданные для документации.

Ответы от API: форматы данных

FastAPI по умолчанию возвращает ответы в формате JSON. Вы можете возвращать из функции-обработчика стандартные типы Python, и FastAPI автоматически преобразует их:

  • dict -> JSON object {}
  • list -> JSON array []
  • str, int, float, bool -> JSON string, number, boolean

Использование response_model.
Для обеспечения строгой типизации ответа API, можно использовать аргумент декоратора response_model. Он принимает Pydantic-модель и гарантирует, что ответ будет соответствовать ее структуре (отфильтрует лишние поля, преобразует типы).

from fastapi import FastAPI
from pydantic import BaseModel
from typing import Optional

class UserOut(BaseModel): # Pydantic модель для ответа
    id: int
    username: str
    email: Optional[str] = None

app = FastAPI()

@app.get("/users/{user_id}", response_model=UserOut)
def get_user(user_id: int):
    # Предположим, мы получили из БД объект с лишними полями (например, хэш пароля)
    user_from_db = {
        "id": user_id,
        "username": "alex",
        "email": "alex@example.com",
        "hashed_password": "very_secret_hash_value"
    }
    # FastAPI отфильтрует ответ согласно UserOut, и hashed_password не попадет в JSON
    return user_from_db

Это также улучшает автоматическую документацию, показывая точную схему ответа.

Встроенная документация с OpenAPI (Swagger)

Одним из главных преимуществ FastAPI является автоматическая генерация интерактивной документации API.

Как это работает:

  • FastAPI анализирует ваш код: пути, декораторы операций (@app.get), параметры функций, их аннотации типов, Pydantic-модели.
  • На основе этого анализа он генерирует схему API в соответствии со стандартом OpenAPI.
  • Эта схема используется для создания двух вариантов интерактивной документации, доступных в браузере.

URL документации (по умолчанию):

  • /docs: интерактивный интерфейс Swagger UI. Позволяет не только просматривать документацию, но и тестировать эндпоинты прямо из браузера.
  • /redoc: альтернативный интерфейс ReDoc. Предоставляет чистую и легко читаемую документацию (без возможности отправки запросов).

Пример минимального приложения FastAPI

Этот пример объединяет несколько рассмотренных концепций.

Файл main.py:

from fastapi import FastAPI, Query
from pydantic import BaseModel
from typing import Optional, List

# --- Pydantic модель для ответа ---
class Item(BaseModel):
    id: int
    name: str
    is_offer: Optional[bool] = None

# --- Экземпляр FastAPI ---
app = FastAPI(
    title="Мое Первое API",
    description="Это демонстрационный API на FastAPI",
    version="1.0.0",
)

# --- Данные (заглушка вместо БД) ---
db_items = {
    1: {"id": 1, "name": "Яблоко"},
    2: {"id": 2, "name": "Банан", "is_offer": True},
    3: {"id": 3, "name": "Апельсин"},
}

# --- Эндпоинты ---
@app.get("/", summary="Корневой эндпоинт")
async def read_root():
    return {"message": "Добро пожаловать в FastAPI!"}

@app.get("/items/", response_model=List[Item], summary="Получить список товаров")
async def get_items(limit: int = Query(10, gt=0, le=100)):
    return list(db_items.values())[:limit]

@app.get("/items/{item_id}", response_model=Item, summary="Получить товар по ID")
async def get_item_by_id(item_id: int):
    return db_items.get(item_id)

Запуск: uvicorn main:app --reload.
Тестирование: откройте http://127.0.0.1:8000/docs в браузере.
